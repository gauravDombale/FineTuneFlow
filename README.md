**FineTuneFlow** is an end-to-end LLM alignment pipeline specifically engineered to run efficiently on constrained Apple Silicon hardware (like the M2 with 8GB RAM). 

This project transforms a base Instruct model into a strict **Tool-Calling Agent**. It employs a multi-stage training process—starting with **Supervised Fine-Tuning (SFT)** via QLoRA, followed by **Direct Preference Optimization (DPO)**—to perfectly align the model's outputs to specific JSON schemas without hallucinating conversational text.

---

## 🎯 Architecture & Pipeline

This repository is built around a sequence of heavily automated data generation, evaluation, and training loops using Apple's `mlx-lm` framework.

1. **Synthetic Data Engine (`scripts/prepare_data.py`)**  
   Automatically generates perfectly formatted ChatML datasets mimicking the Glaive tool-calling structure to bypass network constraints and guarantee high-quality training pairs.
2. **SFT via QLoRA (`config_sft.yaml`)**  
   Teaches the base model to output strictly formatted `{"name": "...", "arguments": {...}}` JSON strings. 
3. **Automated Evaluation Engine (`scripts/eval_baseline.py`)**  
   Tracks Parse Rate, Name Accuracy, Argument F1, and Exact Matches across Zero-Shot, SFT, and DPO stages.
4. **Direct Preference Optimization (`scripts/train_dpo.py`)**  
   Leverages corrupted/"rejected" examples generated by `scripts/prepare_dpo_data.py` to heavily penalize structural hallucinations and enforce correct syntax generation.
5. **Inference Server (`api/main.py`)**  
   A lightweight `FastAPI` server that lazy-loads the best trained adapter weights for real-time tool-call generation.

## 🚀 Getting Started

### Prerequisites
- macOS (Apple Silicon M1/M2/M3)
- Python 3.10+
- [`uv`](https://github.com/astral-sh/uv) (Extremely fast Python package installer)

### Quick Setup

The entire workflow is managed through a simple `Makefile`. Run the commands sequentially:

#### 1. Bootstrap Environment & Data
```bash
make setup      # Creates the virtual environment and installs dependencies via uv
make data       # Generates the SFT training dataset
make dpo-data   # Generates the DPO preference dataset
```

#### 2. Training Workflow
```bash
make prompt-baseline  # Evaluates the zero-shot base model performance
make baseline

make sft              # Executes QLoRA Supervised Fine-Tuning
make sft-eval         # Evaluates SFT performance
make analyze          # Dumps error logs and parses failures

make dpo              # Executes DPO training against rejected data
make dpo-eval         # Evaluates final DPO performance
```

#### 3. Reporting & Demo
```bash
make report           # Generates a Markdown ablation study comparing metrics
make demo             # Launches the FastAPI server at http://localhost:8000
```

## 🔌 API Usage

Once `make demo` is running, you can test the model's tool-calling capabilities via a simple `POST` request.

**Request:**
```bash
curl -X POST http://127.0.0.1:8000/tool_call \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Can you book a flight from NYC to London for tomorrow?",
    "tools": [{
      "name": "book_flight",
      "description": "Book a commercial flight.",
      "parameters": {
        "type": "object", 
        "properties": {
          "origin": {"type": "string"},
          "destination": {"type": "string"}
        }
      }
    }]
  }'
```

**Expected Response:**
```json
{
  "name": "book_flight",
  "arguments": {
    "origin": "NYC",
    "destination": "London"
  }
}
```

## 📊 Evaluation Metrics

The pipeline outputs an ablation report (`report/REPORT.md`) tracking the metric progression:
- **Parse Rate**: Percentage of outputs that successfully parse via `json.loads()`.
- **Name Accuracy**: Percentage of outputs that call the correct tool.
- **Argument F1**: Structural match of the requested vs generated JSON keys.
- **Exact Match**: Perfectly formed tool calls.

*By the end of the DPO phase, the Exact Match rate approaches near-perfect performance on the test split.*
